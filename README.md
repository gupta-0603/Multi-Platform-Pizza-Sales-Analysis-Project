# 🍕 Pizza Sales Analysis - Multi-Platform Data Project

This project focuses on a comprehensive, end-to-end data analysis of pizza sales performance. To demonstrate versatility across the modern data stack, the analysis, data cleaning, and visualization workflows have been fully executed across four different platforms: **Power BI, Python (Jupyter Notebook), Excel, and Tableau**.

---

## 📌 Project Workflow
Data Source (.csv) ➡️ Data Cleaning & Exploration (Python) ➡️ Multi-Platform Visualizations & Dashboards (Power BI | Tableau | Excel)

---

## 🔧 Platform Implementations

### 1. Python (Jupyter Notebook)
* **Libraries Used:** `pandas`, `numpy`, `matplotlib`, `seaborn`
* **Steps Taken:** Implemented data cleaning, handled missing values, standardized date-time formats, and executed Exploratory Data Analysis (EDA) to find initial correlations between peak hours and order volumes.

### 2. Power BI Dashboard
* **Techniques:** Engineered custom DAX measures for critical KPIs (Revenue, Order counts), built a dedicated Date table for Time Intelligence, and optimized interactive filtering.
* **Key Visuals:** Hourly/Monthly sales trends, sales by pizza category, and top/bottom selling pizzas.

### 3. Excel Interactive Dashboard
* **Techniques:** Utilized advanced Power Query functions for structural transformation, designed dynamic Pivot Tables, and introduced Slicers for an interactive executive view.

### 4. Tableau Dashboard
* **Techniques:** Created calculated fields, configured dual-axis charts for hybrid trend tracking, and formatted custom tooltips to enhance user interactivity.

---

## 🧠 Key Insights & Inferences
* **Peak Hours:** Highest order volumes occur during lunch hours (12:00 PM - 1:00 PM) and dinner hours (5:00 PM - 8:00 PM).
* **Top Performers:** Classic and Supreme category pizzas generate the highest percentage of total revenue.
* **Size Preferences:** Large (L) size pizzas contribute significantly more to the overall revenue compared to Regular (R) or Extra Large (XL) variants.
